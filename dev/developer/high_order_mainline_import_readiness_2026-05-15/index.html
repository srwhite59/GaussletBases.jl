High-Order Mainline Import Readiness

Date: 2026-05-15

Purpose

This memo identifies the stable slice of the high-order lane that is ready for repo-manager to consider importing into mainline soon. The import candidate is not the full high-order/OPCU research program. It is the dimension-organized owned-unit formulation needed to replace unsplit shared rectangular molecular shells by endcap/panel units in the ordinary H2 nested construction.

The strongest current evidence is the old-standard-aligned H2 route: it keeps the old parent basis, molecular support, and split atomic child/core content, then replaces only unsplit shared molecular shell layers. That makes the comparison local to the shell ownership policy instead of changing the whole molecular basis.

Definitions

  • Owned unit: a retained contraction unit with a declared support region, a contraction map, and an assembly role.
  • Support mask: the set of parent grid directions owned by a unit. Importable endcap/panel units require exact support coverage with no duplicate owners inside a replaced shell.
  • Contraction map: the coefficient map from owned support directions to retained basis functions. For the current import candidate these maps remain IDA-safe and have positive contracted weights.
  • Assembly role: whether a unit is part of an unsplit shared molecular shell, a split atomic child/core region, or another preserved inner component.
  • Old shared molecular shell: an ordinary nested rectangular shell layer that remains one shared molecular object before the construction splits into separate atomic child/core boxes.
  • Split atomic child/core content: the existing ordinary nested inner H2 chemistry content after the molecular source has split. This content is left unchanged by the import candidate.
  • Endcap/panel shell: a replacement owned-unit shell made from two endpoint caps and four side panels. The side panels absorb the four long edge lines, so ownership is disjoint.
  • q: the retained one-dimensional order floor used on each cap/panel transverse direction.
  • L: the retained long-direction length used by each side panel.
  • Finite IDA Hamiltonian ED: exact diagonalization of the finite Hamiltonian generated by the current IDA operator route. The caveat is the IDA interaction representation and finite basis, not that the ED solve is somehow not exact for its finite H,V.

Import Candidate

The stable candidate is a small internal producer path for replacing every unsplit shared rectangular molecular shell by endcap/panel owned units:

retained count = 2*q^2 + 4*q*L

The producer should preserve the old shell support exactly, keep ownership disjoint, and leave split atomic child/core boxes untouched. It should not expose a broad public OPCU framework yet.

The current recommended default for stretched H2 is all-shared q=4,L=4. This replaces each unsplit shared molecular shell with two 4 x 4 caps and four 4 x 4 side panels. It is the best current balance between count control and energy stability. The q=3,L=3 policy is useful as a low-count stress case but was too aggressive when all shared shells were replaced. The q=5,L=5 policy improves R = 4 energy slightly but costs substantially more functions.

What Should Stay Experimental

Do not import the following as part of this mainline slice:

  • Atom-centered high-order FSB/FBU work.
  • Distorted raw-cube completeness or distorted local-cube claims.
  • Arbitrary projected residual shell/core repairs. These broke current IDA fixed-block packet assumptions and should not be normalized.
  • The broad OPCU framework machinery as a public abstraction.
  • Scratch external-target studies, Gaussian-target probes, D-shell target builders, and side-wall usefulness tests.
  • Any claim that the endcap/panel H2 evidence proves production high-order nesting, correction-layer usefulness, DMRG behavior, or larger-molecule transferability.

Those threads remain useful research context, but the mainline-ready slice is only the owned-unit producer for old-standard-aligned shared rectangular shell replacement.

Proposed Mainline Shape

Implement this as a small internal or experimental producer path, not as a new public framework. A reasonable shape is:

  • An internal owned-unit specification for rectangular molecular shell layers.
  • A support-mask checker that verifies exact coverage, no duplicate ownership, and no cap/panel overlap.
  • An endcap/panel contraction-map producer parameterized by q and L.
  • A guarded insertion point that applies the producer only to unsplit shared molecular shell layers and leaves split atomic child/core boxes unchanged.
  • Internal names that make the status explicit, for example experimental_endcap_panel_shared_shell or similar.

The implementation should reuse the existing ordinary nested fixed-block and operator machinery. It should not introduce arbitrary post-projection residual columns or nonpositive contracted weights.

Test Ladder

The import test ladder should be incremental:

  1. Anatomy and support ownership: list every candidate shared shell, confirm it is a single rectangular molecular shell, and verify no missing or duplicate owned support directions.
  2. Fixed-block/operator preflight: verify fixed overlap, positive contracted weights, finite gaussian_sum and pair_sum, operator overlap, and H/V symmetry.
  3. R = 4 H2 S/P HF/ED: compare old S/P ordinary against all-shared q=4/L=4 with the same finite IDA Hamiltonian and the same HF/ED route.
  4. Stretched H2 regression: run R = 2.5, 3.0, 4.0, 5.0, 6.0 and join against old S/P ordinary. Track dimension, ED total, ED-HF lowering, BO error where references are available, and residuals.

Passing this ladder would make the internal endcap/panel producer ready for mainline experimental use. It would not make the broad high-order lane or OPCU framework production-ready.

Import Recommendation

Repo-manager can consider importing the owned-unit/endcap-panel shared-shell producer soon, with q=4,L=4 as the current default for stretched H2 tests. The import should be explicitly experimental/internal and should include the test ladder above. The broader high-order lane should remain out of mainline until its objects and validation targets are narrowed to similarly stable slices.
